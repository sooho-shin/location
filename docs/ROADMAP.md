# LOCATION 로드맵

## 문서 목적

이 문서는 LOCATION 프로젝트의 개발 방향을 단계별로 정리한다. 실제 구현 순서는 사용자 요청과 우선순위에 따라 바뀔 수 있지만, 큰 방향은 이 문서를 기준으로 조정한다.

## 현재 단계

현재 프로젝트는 로컬 동작 가능한 MVP 단계다.

완료된 기반:

- Next.js 프론트엔드
- Express 백엔드
- Google Maps JavaScript API 지도 표시
- Gemini API 기반 추천
- Google Places API 후보 검색과 상세 조회
- 카테고리 선택 바텀시트
- 지도 마커와 클러스터 선택
- 장소 상세 패널

## 1단계: 안정화

목표: 로컬 개발과 기본 사용 흐름을 안정화한다.

- 지도 로딩 실패 상태 정리
- API 키 누락 상태 안내 개선
- 추천 실패 메시지 개선
- 위치 권한 거부 상태 화면 추가
- Google Places quota 또는 결제 오류 대응 문구 정리
- 프론트엔드 빌드와 백엔드 빌드 검증 루틴 확정

## 2단계: 제품 UX 정리

목표: 모바일 사용자가 실제로 탐색하기 좋은 앱으로 다듬는다.

- 바텀시트 상태 확장
- 추천 결과 목록과 지도 마커 연동 강화
- 장소 상세 패널 정보 구조 개선
- 클러스터 선택 패널 모바일 위치 보정
- 카테고리 이미지와 설명 품질 개선
- 로딩 상태와 skeleton UI 정리

## 3단계: 다국어와 일본인 타겟 강화

목표: 일본인 사용자를 우선 타겟으로 한 UX를 제품 기능으로 만든다.

- UI 문자열 분리
- 한국어, 일본어, 영어 언어 정책 확정
- 카테고리명과 추천 키워드 다국어화
- Gemini 응답 언어 제어
- 일본인 관광객 기준 카테고리 확장

## 4단계: 추천 품질 개선

목표: 추천 결과의 정확도와 설명 품질을 높인다.

- 카테고리별 검색 반경과 키워드 조정
- 후보 장소 필터링 규칙 추가
- 영업 중 여부와 평점 기반 정렬 검토
- 중복 장소 제거 개선
- 추천 이유와 방문 팁 분리

## 5단계: 운영과 배포

목표: 외부 사용자가 접근 가능한 서비스로 배포한다.

- 프론트엔드 Vercel 배포
- 백엔드 Railway, Render, Fly.io, Docker 중 선택
- CORS 정책 정리
- API 키 제한과 billing 상태 확인
- 배포 환경 변수 문서화
- 기본 모니터링과 에러 로그 정리

## 보류 항목

다음 항목은 기획 확정 전까지 임의로 구현하지 않는다.

- 사용자 계정과 로그인
- 즐겨찾기 영구 저장
- 결제 기능
- 관리자 페이지
- 자체 장소 데이터베이스
- 추천 기록 저장
- 소셜 공유 고도화
