# 프로젝트 작업 규칙

## 기준 문서 우선순위

- 이 프로젝트에서 코드, 문서, 디자인, 배포 설정을 수정하기 전에는 `docs/PRODUCT_SPEC.md`를 먼저 확인한다.
- API 요청/응답, 데이터 모델, 외부 API 연동을 바꾸는 작업은 `docs/API_CONTRACT.md`를 함께 확인하고 필요한 경우 갱신한다.
- 구현 방식, 환경 변수, 검증 방법, 금지 사항은 `docs/DEVELOPMENT_GUIDE.md`를 따른다.
- 기능 우선순위나 단계별 개발 방향을 판단할 때는 `docs/ROADMAP.md`를 참고한다.
- 기획 문서와 코드가 충돌하면 사용자가 명시적으로 구현을 우선하라고 하지 않는 한 기획 문서를 먼저 갱신하거나 사용자에게 확인한다.

## 문서 언어

- 이 프로젝트의 Markdown 문서는 사용자가 다른 언어를 명시하지 않는 한 한국어로 작성한다.
- 새 Markdown 문서를 만들거나 기존 Markdown 문서를 수정할 때 제목, 설명, 체크리스트, 주석성 문구를 한국어로 유지한다.

## 커밋 메시지

- 이 프로젝트의 커밋 메시지는 항상 한국어로 작성한다.
